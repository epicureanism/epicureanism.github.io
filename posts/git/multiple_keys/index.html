SourceTree with Multiple SSH Keys

Jun 21, 2021 · 1 min read

To setup multiple ssh keys usage in SourceTruee on Windows.

Activate OpenSSH Agent

By default, this agent is not activated, and ssh-add will not work.

  • Win + R, type
  services.msc
  • Find “OpenSSH Authentication Agent”, activate it.

Edit ~/.ssh/config

To specify multiple key configurations in the config of .ssh in the user home folder.

# Same Host and HostName means the default setting of the HostName
Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/user1

# Another key with customized Host but with the same HostName
Host epi
  HostName github.com
  User git
  IdentityFile ~/.ssh/user2

Change the git url for repo that uses the another key

git@epi:user/repo.git

SourceTree

  • Tools > options > General
    • Change the SSH client to “OpenSSH”
    • Specify different ssh keys with “;” seperated

    C:\Users\user\.ssh\user1;C:\Users\user\.ssh\user2
